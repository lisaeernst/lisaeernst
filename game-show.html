Project Breakdown

More fun than a barrel of monkey's.
But better... coding monkey's!

In this project the HTML and CSS was pre-written and provided to me by Treehouse. The goal in this project was for me to interact with the HTML elements along with creating new elements dynamically inside of my Javascript.

The user must guess and select a letter from the on screen keyboard. The logic had to be created to compare what the user selected against the letters in the randomly created phrase. If the user guesses one of the letters in the phrase that letter is then displayed on the screen. They have five hearts or chances to make a mistake, if they go over the five chances they loose and are shown a new screen with Sorry, please try again.

I like to make extensive comments in my code to explain what each step is accomplishing. This has allowed me to go back into my code later, such as when writing these case studies for my portfolio, to understand quickly the logic and reasoning behind each decision I made.

The Game Page

Solving the random phrases using the on screen keyboard

getElementById

Storing variables

To start off, I used getElementById to store each ID in my HTML document and stored them each in a variable. I created a variable named ‘missed’ and gave it a value of zero so as the user selects a letter on the keyboard each wrong selection is then stored in this variable up to a maximum of 5 chances. After that the user is shown a Sorry, please try again message.

I then created an addEventListener set on the event ‘click’, to respond when the user clicks on the Start Game button located on the home screen. This then hides the overlay div by setting its display to ‘none’, consequently displaying the game screen.

I created an array named ‘phrases’ to hold my five phrases. Then I created a function named getRandomPhraseAsArray and inside that function I created a variable that stores a random number based on the length of the array.

The addPhraseToDisplay function loops through the array of characters from the getRandomPhrasesAsArray function previously created. It then puts the characters inside the the html document li items and gives them a class of ‘letter’.

The next step was to create a variable to hold all of the letters with the class of letter and convert them to lower case, then check to see if the letter matches any of the letters in the phrase and if so, it gets stored in the matched variable with a class of show. When the loop is completed then the matched variable is returned.

Next, I added an event listener for the qwerty element that listens for the click event on the on screen keyboard with a tag name of button this checks to make sure the user is not using the physical keyboard. The class of chosen is then added to the button that was clicked. That key is then disabled and is not available to be chosen again and is highlighted on the onscreen keyboard using CSS.

The checkLetter function is called and the results are stored in the match variable. If the checkLetter function does not have a valid value then match is equal to null and a heart is removed, giving the user one less chance to solve the phrase. The missed counter is incremented by one.

A new function called checkWin was created and inside this function the letters and show variables are compared to see if their lengths match. If they are then the win overlay is displayed. If the two variables lengths do not match then display the lose overlay with a message saying, Sorry, please try again.

I added a function named reset to create a reset button which reloads the page on both the win or lose overlay to make for better usability for the player.
