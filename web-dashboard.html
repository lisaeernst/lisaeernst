Project Breakdown

More fun than a barrel of monkey's.
But better... coding monkey's!

This was my seventh Treehouse project which includes three charts using the Charts.js library, social media widgets, a new members and recent member activity section, a form with an autocomplete feature to search for a user and then message users on the system, and the use of localstorage to save settings such as email and profile preferences and their time zone selection.

In the header there is an alert drop down modal that shows recent member activity as well as the current logged in users photo and name.

As with most Treehouse projects I was given a mockup of how the site should look when completed. This included desktop, mobile, and tablet views for responsive design/development. Using semantic markup I created the static site using html, css, and css grid knowing that these areas would then be dynamically injected with content using Javascript.

Treehouse had a great section on creating, using, and working with svg’s in this part of the course. Consequently, this project then made extensive use of svg’s and I learned a lot by implementing them in this project.

Some of the challenges with this project were in applying the charts.js library as this was the first time I had used a library like this before.

Other areas that were challenging were getting the design pixel perfect using the grid, getting localstorage to work by remembering the users preferences in the Settings section at the bottom of the app (this wasn’t a requirement to pass but I really wanted this to work by using the Save and Cancel buttons), and overall the functionality of the Message User form with a Javascript widget called awesomplete.

This search feature also wasn’t required but I wanted it to work. So, in the end the app works as intended and a bit beyond and I was happy with the end result.
